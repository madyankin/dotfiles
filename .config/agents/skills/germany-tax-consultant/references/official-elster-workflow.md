# Official ELSTER Workflow

Use this reference for questions about filing through ELSTER, account setup, login methods, and portal wording.

## Official Sources

- `https://www.elster.de/eportal/start`
- `https://www.elster.de/eportal/helpGlobal?themaGlobal=help`

## What To Use This For

- Explain what ELSTER is and when it is the correct filing channel.
- Explain registration, account activation, and certificate-based access.
- Explain app-based login and `ElsterSecure` only at a high level unless the user asks for setup details.
- Explain prefilled return features and how third-party data can appear in the return.
- Explain practical navigation wording users are likely to see in German.

## Guidance

- Prefer official ELSTER terminology when referring to screens, buttons, and account features.
- Explain in English first, then add the German term in parentheses on first mention.
- Distinguish clearly between:
  - account creation,
  - authentication method,
  - completing forms, and
  - submission.
- If portal wording changes, avoid pretending exact labels are guaranteed. Say that menu names can change slightly.

## Common Topics

- `Benutzerkonto` / user account
- certificate file / `Zertifikatsdatei`
- `ElsterSecure`
- submission / `Versenden`
- notices / `Bescheide`
- prefilled data / `Bescheinigungen`

## Response Pattern

For ELSTER help:

1. Identify whether the user needs account setup, login recovery, form-filling help, or submission help.
2. Ask for the tax year and whether an ELSTER account already exists.
3. Give step-by-step guidance using bilingual terminology.
4. Flag when the exact screen name may differ.
5. If blocked by identity verification, account recovery, or certificate loss, direct the user to official ELSTER help rather than guessing.

## Limits

- Do not claim that a specific screen layout is current unless the official help page confirms it.
- Do not invent troubleshooting paths for account recovery.
- Do not advise unsafe workarounds for identity or authentication issues.
